# 계획 문서 작업 흐름

`docs/plans`는 사용자 확인 또는 실행을 기다리는 구현 직전 단계의
Markdown 계획 문서를 모아두는 위치다. 이 문서들은 실제 작업을 시작하기
직전의 인계 지점으로 취급한다.

## 활성 계획

- 대기 중이거나 진행 중인 계획은 `docs/plans` 바로 아래에 둔다.
- 파일명은 날짜 접두사와 kebab-case를 사용한다. 예:
  `2026-06-07-better-auth-google-social-login.md`.
- 각 계획 문서는 해당 작업 범위의 기준 문서로 취급한다.
- 새 계획 문서 상단에는 아래의 필수 보관 규칙 문구를 포함한다.
- 문서에 적힌 모든 필수 작업이 완료되고 검증되기 전에는 해당 계획을
  `archive`로 옮기지 않는다.

새 계획 문서에 포함할 필수 보관 규칙:

```markdown
> **보관 규칙:** 이 계획의 모든 작업을 완료하고 검증한 뒤, 최종 응답을
> 보내기 전에 이 파일을 `docs/plans/archive/`로 옮긴다.
```

## 완료 및 보관 규칙

에이전트가 활성 계획의 모든 항목을 완료하면, 작업을 끝내기 전에 해당
계획 문서를 `docs/plans`에서 `docs/plans/archive`로 자동 이동해야 한다.

이동할 때는 원래 파일명을 유지한다.

```text
docs/plans/<plan-file>.md
docs/plans/archive/<plan-file>.md
```

완료된 계획만 `docs/plans/archive`에 둘 수 있다. 계획이 아직 차단된 상태,
부분 완료 상태, 사용자 입력 대기 상태, 또는 의도적으로 보류된 상태라면
`docs/plans`에 그대로 둔다.

## 에이전트 체크리스트

계획을 바탕으로 한 작업을 최종 완료하기 전에 다음을 확인한다.

- 계획의 모든 작업과 검증 단계가 완료되었는지 확인한다.
- 계획에서 요구한 후속 작업이 unresolved 상태로 남아 있지 않은지 확인한다.
- 완료된 계획 파일을 `docs/plans/archive`로 이동한다.
- 최종 응답에서 계획 문서를 archive로 이동했음을 언급한다.
